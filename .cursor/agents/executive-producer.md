# Executive Producer

**Role ID:** `executive-producer`  
**Primary owner:** Oscar  

## Responsibilities

- Project vision and final design decisions  
- Mission priority (“best next ≠ biggest”)  
- **High-level Mission Director commands only** — do not manually orchestrate Skills  
- **Visual Design Package approval** (mental play) before Ready to Implement on production slices  
- Human Gameplay validation (Enhanced Input PIE)  
- Written waivers when proceeding on unvalidated foundations or (rare) VDP skip  
- Milestone approval, Ready / merge decisions  
- GitHub repository ownership  

## Preferred commands

```text
Start Mission PE-###
Continue Mission PE-###
Generate Visual Package PE-###
Implement Mission PE-###
Validate Mission PE-###
Review Mission PE-###
Close Mission PE-###
```

Mission Director selects Skills automatically (`Documents/04_Production/AIStudio/MissionDirector/`).

## Boundaries (do not own)

- Day-to-day Blueprint graph authoring (Lead Developer / implementer)  
- Writing Story Bible prose (Narrative Director)  
- Lighting art direction details (Technical Artist / Horror)  
- Authoring playtest steps only — EP **executes** Gameplay PASS  
- Authoring Previs artifacts — Previs skills produce the VDP; EP **evaluates** it  
- Picking which Cursor skill runs next — **Mission Director** owns orchestration  

## Inputs

- Design plans, **Visual Design Packages**, completion reports, PRB verdicts  
- Mission Director gate STOP reports  
- Manual PIE checklists  
- Roadmap / health  

## Outputs

- High-level commands  
- Design Plan proceed / reject  
- **VDP APPROVE / RETURN TO DESIGN** (sets Ready to Implement)  
- Approve / Reject / Waiver  
- Gameplay PASS / FAIL / PENDING  
- Merge authorization  

## Visual Design Package gate (permanent)

Before Ready to Implement on production missions with a player spatial loop, answer Yes/No:

1. Can I understand the mission?  
2. Can I mentally walk through the level?  
3. Can I understand every objective?  
4. Can I understand every puzzle?  
5. Can I understand the pacing?  
6. Can I imagine the horror?  
7. Can I identify confusing areas?  
8. Would I enjoy playing this?  

Any **No** → return to Design / Previs. Do not approve text alone — approve the **player experience**.
