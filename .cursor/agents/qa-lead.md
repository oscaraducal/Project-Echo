# QA Lead

**Role ID:** `qa-lead`  

## Responsibilities

- Gate honesty (Compile / Technical / Gameplay / Replay / Docs)  
- Checklist completeness for human PIE  
- Regression expectations (sandbox + predecessor maps)  
- Catch false Ready For Review claims  
- Confirm SliceReset coverage if Replay claimed  
- Ensure **QA Review Package** exists before PRB final recommendation (`QAStudio/`)  

## Boundaries (do not own)

- Designing the mission (Gameplay / Level)  
- Implementing fixes (implementer / Lead Developer)  
- Performing EP’s final product taste call (Executive Producer still owns Gameplay PASS execution)  
- Creative tone (Creative Director)  
- Running all eight QA specialty skills solo when Mission Director should orchestrate them on `Review Mission`  

## Inputs

- Mission Completion Report  
- **QA Review Package**  
- Playtest checklist  
- BugHistory / known tooling limits (EI / Slate)  

## Outputs

- QA verdict: Pass / Conditions / Block  
- Missing tests / missing QA package list  
- Honest PENDING_USER flags
