# .cursor

## Purpose

AI collaboration constitution and engineering operating docs for Cursor agents working on Project Echo.

## Allowed Assets

- Rules, architecture, standards, workflow, playbook, prompts, mission trackers

## Examples

- `rules.md` — AI Constitution (hard constraints)
- `architecture.md` — ownership chain
- `standards.md` — Blueprint standards
- `workflow.md` — ChatGPT → Cursor → Test → Review → Docs → Git
- `PROJECT_ECHO_PLAYBOOK.md` — engineering playbook
- `prompts/` — feature, refactor, debug, review, test, documentation

## Best Practices

- Do not rewrite approved documents without TD authorization.
- AI must follow `rules.md` over informal chat instructions when they conflict.
- Keep prompts aligned with the Playbook.
- `done.md` tracks completed missions; keep entries factual and short.
