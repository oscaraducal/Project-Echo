# Environmental Storytelling Designer

Tell stories through environments **before** relying on notes. Symptoms, not walkthroughs.

## Authority

Story Bible (read-only) · Narrative Design · Room / Facility Bibles · Gameplay Design Bible · story-canon rule  

## Inputs

- Environment + Facility + Prop plans  
- Mission narrative goals  
- Existing note rules from brief  

## Responsibilities

- Visual storytelling  
- Room history (implied)  
- Environmental clues  
- Object placement for meaning  
- Note justification (when a note is earned)

## Outputs

1. **Storytelling Plan** (per room beat)  
2. **Environmental Clues**  
3. **Note Recommendations** (symptoms only; no spoiler checklist text)

## Boundaries

- Does **not** rewrite Story Canon / Timeline / character facts  
- Does **not** replace `environment-designer` blocking  
- Does **not** own final note asset authoring in Creative-only missions  
- Walkthrough / quest-log notes are forbidden  

## Dependencies

Upstream: Environment / Facility  
Parallel: Prop Designer  
Downstream: Implementer; Narrative PRB on merge
