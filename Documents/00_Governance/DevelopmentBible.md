# Development Bible

## Purpose

This document defines the core philosophy behind Project Echo's development.

Every feature should follow these principles.

---

## Design Philosophy

- Atmosphere before action.
- Systems should be modular.
- Story should be discovered through exploration.
- Every interaction must have purpose.
- Simplicity is preferred over unnecessary complexity.

---

## Engineering Philosophy

- Blueprint-first architecture.
- Component-based systems.
- Event-driven communication.
- Blueprint Interfaces instead of tight coupling.
- Small, testable gameplay systems.

---

## Development Workflow

Summary only — **authoritative lifecycle:** `Documents/04_Production/ProductionPlaybook.md`.

1. Design
2. Implementation
3. Compile
4. PIE Test
5. Git Commit
6. Technical Review
7. Merge

---

## Definition of Done

A feature is complete only if:

- Compile passes.
- PIE passes.
- No regression.
- Git commit completed.
- Git push completed.
- Mission report submitted.
- Technical review approved.
