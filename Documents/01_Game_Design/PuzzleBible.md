# Purpose

This document defines the design philosophy for puzzles in Project Echo.

Puzzles should feel like natural problems within the environment rather than artificial game challenges.

---

# Puzzle Philosophy

Every puzzle should answer one question:

"Why would this exist in a real research facility?"

If the answer is unclear, redesign the puzzle.

---

# Puzzle Types

Environmental

- Restore power
- Unlock security systems
- Redirect electricity

Observation

- Read notes
- Interpret clues
- Follow maintenance procedures

Exploration

- Locate missing items
- Discover hidden rooms
- Find alternate routes

Combination

- Multiple systems working together

---

# Difficulty

Players should solve puzzles through observation.

Avoid:

- Random guessing
- Trial and error
- Pixel hunting

The environment should always provide clues.

---

# Clues

Clues may appear through:

- Notes
- Whiteboards
- Room layout
- Warning labels
- Maintenance logs
- Audio messages

Important clues should be discoverable but not forced.
