# 07_Changelogs

## Purpose

Human-readable changelogs for prototypes, vertical slice builds, and releases.

## Allowed Assets

- Changelog markdown/text files

## Examples

- Future: `CHANGELOG_0.1.0.md`

## Best Practices

- Summarize player-facing and engineering-facing changes separately when useful.
- Prefer appending over rewriting history.
- Align version numbers with `DefaultGame.ini` ProjectVersion when cutting a milestone.
