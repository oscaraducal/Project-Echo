# 09_Engineering

## Purpose

Engineering writeups for implemented systems: purpose, architecture notes, variables/functions summaries, test notes, limitations.

## Allowed Assets

- Markdown (preferred for AI/review) or approved engineering docs
- Per-feature implementation notes

## Examples

- Future: `Interaction_System.md`, `Flashlight_System.md`

## Best Practices

- Every feature mission should leave a short engineering note here (or as directed by the Playbook).
- Do not store Canon/Narrative design here — that belongs in `01_Game_Design/`.
- Link to Blueprints by asset name/path, not by pasting binary content.
